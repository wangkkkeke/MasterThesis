1. 命名规范
1.1 变量命名
- 使用有意义的描述性名称
- 局部变量使用小写字母
- 全局变量使用g_前缀
- 指针变量使用p前缀
- 常量使用全大写字母，单词间用下划线连接
例如:
int count;
float g_totalAmount; 
char* pName;
const int MAX_SIZE = 100;

1.2 函数命名
- 使用动词+名字
- 采用驼峰命名法
- 函数名应清晰表明其功能
例如:
void calculateTotal();
int getUserInput();
float computeAverage();

1.3 结构体命名
- 使用大写字母开头
- 采用驼峰命名法
例如:
struct StudentInfo {};
typedef struct _Point Point;

2. 格式规范
2.1 缩进与空格
- 使用4个空格或1个制表符(Tab)缩进
- 运算符两侧加空格
- 逗号后加空格
- 函数参数逗号后加空格
例如:
if (a == b) {
    doSomething();
}

2.2 大括号
- 左大括号不换行
- 右大括号单独一行
- if/for/while等语句必须使用大括号
例如:
while (condition) {
    statement1;
    statement2;
}

2.3 行长度
- 每行代码不超过80个字符
- 超长语句适当换行并对齐

3. 注释规范
3.1 文件注释
- 文件开头注明版权信息、作者、日期等
- 说明文件的主要功能

3.2 函数注释
- 函数声明前添加注释
- 说明函数功能、参数、返回值
- 特殊情况说明
例如:
/*
 * 函数功能：计算两数之和
 * 参数：a - 第一个数
 *      b - 第二个数
 * 返回值：两数之和
 */
int add(int a, int b);

3.3 代码注释
- 关键算法添加注释
- 复杂逻辑说明
- 特殊处理说明
- 使用//进行单行注释
- 使用/* */进行多行注释

4. 编程实践
4.1 错误处理
- 检查函数返回值
- 合理使用断言
- 处理异常情况

4.2 内存管理
- 匹配使用malloc/free
- 防止内存泄漏
- 检查指针是否为NULL

4.3 代码组织
- 相关函数放在一起
- 公共函数放在头文件中
- 适当划分源文件

5. 安全性
- 检查数组边界
- 验证输入数据
- 避免缓冲区溢出
- 注意整数溢出

6. 可维护性
- 避免代码重复
- 保持函数简短，功能单一，参数个数不宜过多（建议不超过5个）
- 函数体不宜过长，建议不超过50行，如果过长应考虑拆分
- 合理使用宏和常量
- 添加必要的注释

7. 编译预处理
- 头文件使用#ifndef防止重复包含
- 合理使用条件编译
- 注意宏定义的副作用

8. 版本控制
- 及时提交代码
- 写明修改说明
- 保持代码版本一致性
