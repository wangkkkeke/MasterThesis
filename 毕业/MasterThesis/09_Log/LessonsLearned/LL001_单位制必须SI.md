# LL001 — 单位制必须全部使用 SI

**日期：** 2026-03（发现），2026-06-30（正式记录）
**来源：** DevelopmentHistory.md Bug002

---

## 教训

inp 文件中材料参数单位混用会导致应力输出差1000倍，且程序不报错、不崩溃，极难发现。

## 错误示例

```
E = 20000  # 单位 MPa — 错误
```

## 正确做法

```
E = 2e10   # 单位 Pa — 正确
```

## 适用范围

所有 inp 文件中的以下参数：
- 弹性模量 E（Pa，不是 MPa）
- 初始地应力（Pa，拉正压负）
- SForceBoundary 返回的力（N，不是 MPa×面积）
- InterfaceBond 刚度 Kn/Kt（N/m）
- 强度参数 ft、c（Pa）
