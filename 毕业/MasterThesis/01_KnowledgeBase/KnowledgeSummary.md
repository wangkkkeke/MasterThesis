## 四、核心方法——预分区局部FEM-DEM共生耦合

### 4.1 基本思想

在同一连续部件（岩石试件、岩柱、岩梁等）内：

```
远场区域          ←→         潜在破裂区
  纯FEM网格                   带键DEM颗粒
  不布置颗粒                  +InterfaceBond
  不进行接触检索              仅此区域检索
         ↑_________界面_________↑
                 InterfaceBond
              (法向/切向弹簧键)
```

**核心优势：** 将颗粒布置与接触检索限制在潜在破裂区，大幅减少远场无效接触计算。

**与全域共生的区别：** 全域共生在整个域内均布置颗粒（接触检索覆盖全域）；本方法仅在破裂区布置颗粒。

**与自适应转化的区别：** 不涉及计算过程中FEM→DEM的在线转化，破裂区在建模阶段预先给定。

### 4.2 InterfaceBond界面键力学模型（本文核心公式）

#### (1) 位移协调与力更新

FEM侧边界颗粒速度由有限元面节点速度插值：

$$\mathbf{v}_c = \sum_{i=1}^{N} w_i\,\mathbf{v}_i \quad \text{（式3-2）}$$

界面力增量更新（增量本构）：

$$\mathbf{F}(t) = \mathbf{F}(t-\Delta t) - \mathbf{K}\,\Delta \mathbf{u} \quad \text{（式3-1）}$$

其中 $\mathbf{K} = \mathrm{diag}(K_t, K_t, K_n)$ 为局部坐标系下接触刚度矩阵。

#### (2) 断裂准则

**拉伸断裂（法向）：**

$$-F_n \geqslant \sigma_t A_{\mathrm{eq}} \quad \text{（式3-3）}$$

**剪切断裂（Mohr-Coulomb）：**

$$F_s \geqslant F_n \tan\varphi + c\,A_{\mathrm{eq}} \quad \text{（式3-4）}$$

#### (3) 键断裂后处理

| 状态 | 法向力 | 切向力 | 强度参数 |
|---|---|---|---|
| 拉伸断裂后 | 归零 | 按残余摩擦修正 | σt、c → 0 |
| 剪切断裂后 | 保留（受压） | 按残余摩擦修正 | σt、c → 0 |
| 断裂后 | 接触算法接管后续相互作用 | — | — |

#### (4) InterfaceBond参数清单

| 参数 | 符号 | 含义 | 配置方法 |
|---|---|---|---|
| 法向刚度 | Kn | 法向弹簧刚度 | 结合FEM网格与DEM颗粒特征尺度预试算标定 |
| 切向刚度 | Kt | 切向弹簧刚度 | 同上 |
| 抗拉强度 | σt | 法向断裂强度 | 以巴西劈裂峰值载荷为目标函数迭代标定 |
| 粘聚力 | c | 剪切断裂粘聚分量 | 同上 |
| 内摩擦角 | φ | 剪切断裂摩擦角 | 同上 |
| 等效截面积 | Aeq | 键承力面积 | 由颗粒间距和投影面积计算 |

### 4.3 显式动力学求解框架（本文核心公式）

系统运动方程：

$$M\ddot{x} + C\dot{x} = F(x) \quad \text{（式3-5）}$$

其中 $F(x)$ = FEM单元内力 + DEM接触力 + InterfaceBond界面键力

时间步稳定性条件：

$$\Delta t \lesssim L_{\min}/c \quad \text{（式3-6）}$$

全局时间步取两子域最小值：$\Delta t = \min(\Delta t_\mathrm{FEM}, \Delta t_\mathrm{DEM})$

### 4.4 稳定性监测指标

| 指标 | 含义 | 正常范围 |
|---|---|---|
| Ghost Force | 界面力残差（非物理附加力） | 相对参考载荷量级可接受（待标定） |
| 系统总能量 | 动能+应变能+界面能 | 应守恒，无异常耗散 |
| 应力波传播 | 波在界面处的透射与反射 | 无异常反射，透射率接近1 |
| 临界时间步 | 各子域时间步比值 | 记录两子域比值，评估失匹配程度 |

### 4.5 DEM区宽度选取原则

基于断裂力学裂纹过程区长度（断裂力学理论下界）：

$$l_p = \frac{E \cdot G_{fI}}{\pi \cdot \sigma_t^2}$$

DEM区宽度须满足：**DEM区宽度 ≥ lp**（理论下界）

实际通过3档宽度（偏窄/合理/偏宽）的敏感性分析验证。

## 七、三个创新点（以终版开题报告docx为准）

### 创新点1：预分区局部共生耦合策略

**问题：** 全域共生模型在远场区域亦维持颗粒网络与接触检索，导致大量无效计算。

**贡献：** 在同一连续部件内，远场仅保留FEM连续描述，仅在潜在破裂区布置带键DEM颗粒，将颗粒布置与接触检索限制在破裂区及其邻域。

**量化验证：** 以接触对数、CPU时间等可量化指标检验减少无效Contact Detection的可行性。

### 创新点2：基于InterfaceBond的异质子域界面协调机制

**问题：** 显式动力学条件下异质子域界面易失稳，跨部件分区耦合以接触传递为主，不适合同一部件内的连续-断裂全过程。

**贡献：** 建立以InterfaceBond为核心的FEM-DEM交界协调机制，统一处理颗粒-颗粒、四面体-颗粒及FEM-DEM交界键的刚度与断裂准则，实现键未断前连续传力、键断裂后局部非连续化的力学过渡。

### 创新点3：局部共生策略对计算效能影响的定量评价

**问题：** 全域共生模型中接触检索规模随计算域增大而快速增长，但缺乏定量对比评价范式。

**贡献：** 构建局部共生与全域共生参照模型，在可比条件下对接触检索规模、计算耗时及力学响应进行比较，分析预分区范围对计算效率与结果精度的影响，为局部共生方法的工程应用提供评价依据。

> 注意：此为终版开题报告docx中的表述。旧版开题报告正文.md中创新点3表述为"多层次验证与评价框架"，已以终版为准。

## 十一、课题来源项目要求（联合协议附件2 + 附件4）

### 11.1 整体项目架构（附件4：技术建议书）

来源：《联合协议附件4：技术建议书》，南京大学拟制，2026年4月

**三层核心架构：**
1. 离散元层：颗粒建模、接触力、GPU并行求解
2. 有限元层：显式动力学、结构应力、大变形计算（四面体统一表示）
3. 耦合层：分区 / 共生 / 自适应转化 三种耦合方式

**共生耦合的具体实现（技术建议书原文）：**
> "在有限元和离散元的界面处采用在位于界面处的有限单元布置**虚拟的离散元颗粒**，该离散元颗粒的位移速度等由其附着的边界有限单元决定，通过虚拟颗粒和离散元颗粒的相互作用实现有限元区域和离散元界面处的相互作用。"

**自适应转化耦合的具体实现：**
- 带拉伸截断的莫尔-库伦本构 + 损伤变量
- 单元损伤超过阈值 → 四面体单元转为球形颗粒集合体
- 转化时质量守恒、速度由单元节点插值、键刚度由E、G确定

**GPU并行策略：**
- 离散元：矩阵离散元（邻居矩阵 + 布尔矩阵），全部在GPU上

- 目标规模：千万级球形颗粒 + 百万级四面体单元协同

### 11.2 待确认关键问题（重要）

**Q_NEW_1：共生耦合实现方式——已分析，部分确认**

**结论：两者描述同一机制，但层级不同，不矛盾。**

完整链路：
```
FEM节点 → 形函数插值 → FEM侧边界代理颗粒 → InterfaceBond(Kn/Kt) → DEM颗粒
     ↑（技术建议书描述此段）              ↑（开题报告描述此段）
```

- **技术建议书（需求/实现层）**：说的是 FEM 边界存在代理颗粒，其运动由有限元节点控制。这是对"上半部分"的描述。
- **终版开题报告（算法层）**：说的是 InterfaceBond 建立 Kn/Kt 弹簧键，FEM 侧颗粒速度由节点插值。这是对"下半部分"的描述。

**第三章推荐表述（来自GPT分析，待导师确认）：**
> FEM区域采用四面体有限元离散，在界面附近建立与有限元位移场绑定的**边界代表颗粒**，其运动学状态由所属有限元节点经形函数插值确定；DEM区域采用可自由运动的离散颗粒描述。InterfaceBond根据几何关系识别界面两侧的FEM边界代表颗粒与DEM边界颗粒，在颗粒之间建立法向与切向键连接，实现跨界面的力与位移传递。因此，InterfaceBond本质上属于**颗粒—颗粒之间的键连接**，而非有限元节点与DEM颗粒之间的直接连接。

**仍需源码确认（Q_NEW_1_残留）**：

代理颗粒在 MultiFracS 源码中的存在形式——三种可能：

| 可能性 | 描述 | 源码特征 |
|---|---|---|
| A（概率最高） | 真实 `Particle` 对象，velocity 由形函数插值赋值 | 能找到 `Particle femParticle` 或 `BoundaryParticle`，以及 `ShapeInterpolation(...)` |
| B | 无对象，每步临时插值后直接计算 BondForce | 无 Particle 对象，只有 `v = N1*v1 + N2*v2 + ...` |
| C | 独立的 `InterfaceNode` 或 `CouplingPoint` 类型 | 既非 Particle 也非 FEM Node，有专用耦合对象 |

**倾向A的三个理由：**
1. 技术建议书反复用"虚拟DEM**颗粒**"，而非"节点"/"插值点"
2. 导师说"本质还是颗粒之间连接"——两端都是颗粒
3. MultiFracS 本身围绕 Particle 构建，额外新建 InterfacePoint 反而更复杂

**源码优先搜索关键词（阅读源码时的路线）：**
```
Particle / VirtualParticle / BoundaryParticle / CouplingParticle
InterfaceBond
ShapeFunction / Interpolation
```
- 若找到 `Particle *fp` 或 `BoundaryParticle` 结构体 → 确认 A
- 若无 Particle，只有 `ShapeInterpolation(...)` 的临时变量 → 确认 B
