## 二、成功方案

### S1. UCS单元验证通过（2026-03）

**场景：** 3D C3D4单元Mohr-Coulomb本构单轴压缩验证

**成功配置：**
- 方案B（无端面压板）：底面仅固定Z向，顶面施加Z向速度，X/Y完全自由
- 方案C（带端面压板）：压板与岩石共节点，接触自然产生端部约束效应
- MonitorElem 移到试样正中心（体心），避免角点应力集中
- 最终理论UCS≈346.41 kPa（c=100kPa, φ=30°），方案B偏差约-4%内

**关键参数：**
```
c=100 kPa, phi=30°, UCS理论 = 2c·cos(φ)/(1-sin(φ)) ≈ 346.41 kPa
```

**教训：** 监测单元不能放在端面或角部，否则端部效应导致应力集中，误判结果。

### S2. FLAC F2D426 圆孔验证 psi=30° 通过（2026-06-15）

**场景：** 四分之一圆环，弹塑性MC介质，孔半径a=1m，外边界R=10m

**成功配置：**
```
材料：G=2.8 GPa, K=3.9 GPa, c=3.45 MPa, φ=30°, ρ=2500
初始应力：σxx=σyy=σzz=-30 MPa（拉正压负）
```

**成功步骤：**

**注意：** psi=30°（关联流）和psi=0°（非关联流）**应力场相同，位移场不同**。

### S4. SForceBoundary施加面力的正确用法（2026-03）

**结论（来自MultiFracS.exe反汇编分析）：**

| 模型维度 | 有效的集合类型 | 无效会报什么错 |
|---------|-------------|--------------|
| 2D | T3D2（线单元集合） | CPS3/CPS4 报 "must be line element set" |
| 3D | 三角形面元集合（surface elements） | C3D4体单元报 "set must be surface elements" |

```
关键错误字符串（从exe提取）：
*SForceBoundary=%s___set error!!!need to be line element set
*SForceBoundary=%s___set error!!!For 2D model, the set must be a line element set
*SForceBoundary=%s The set setting is incorrect!!! 3D model, the set must be a set of surface elements
```

**结论：2D模型SForceBoundary只能用T3D2，不能用CPS3的ELSET。CPS3用于承载面力需经过T3D2中间层。**

### S5. IbkFlag=-10 的正确含义（2026-06-18，来自数据格式文档）

```
IbkFlag=-10 表示：指定区域为连续区域(FEM)，节理单元永不断裂。
用于：实现 FEM+FDEM 的分区耦合，或纯有限元不开裂计算。
特别注意：设置 *PureFDEM=1 时，不能设置 IbkFlag=-10。
```

**不能将 IbkFlag=-10 用于纯弹塑性连续体验证**，否则会插入内部节理单元导致模型刚度改变和数值发散。

## 三、失败方案与原因

### F1. 外边界用位移约束代替应力边界（圆孔验证）

**方案描述：** 外圆弧 r=10 施加固定位移约束（代替 -30 MPa 远场应力）

**结果：** 地应力从自由边界释放，变成"卸载过程"，应力场在边界处不能维持 -30 MPa，无法与FLAC解析解对比。

## 四、踩坑记录（Pitfalls）

### P1. gmsh 点的建立顺序必须逆时针

> "在gmsh中间膜需要从点逆时针建模"

gmsh建立环形区域时，如果点的顺序不是逆时针，Physical Surface的法向方向可能翻转，导致边界条件方向错误（如初始应力方向反号）。

---

### P2. 监测单元必须放在体心，不能放在端面或边界附近

UCS验证中监测单元如果放在试样端面，会受端部约束效应影响，得到的应力明显偏大。必须监测**远离边界的中心单元**。

---

### P3. 显式动力学必须收敛到准静态才能比较

MultiFracS是显式动力学求解器，验证静力问题时必须：
- 确认动能趋近于0（从zmhis.csv的 Ek 列判断）
- 应力不再随时间变化
- 否则提取的应力值无意义

---

### P4. InitSituStress地应力符号约定：压为负

MultiFracS采用"拉正压负"约定，地应力（压应力）必须填写**负值**：
```
*InitSituStress=Volume
1,-30e6,-30e6,-30e6,0,0,0,-7,...
```
如填正值30e6，模型会进入拉伸状态，立即发散。

---

### P5. MC本构开发三个必备DLL

```
Mohr-Coulomb 本构需要同时调用：
1. Ce3D.dll    → Tet_constitutive_equation_increment（线弹性增量）
2. Cjoint3D.dll → Cjoint3D_intact / Cjoint3D_broken（节理损伤与断裂）
3. Mech.dll    → Mech_Ela（应力相关弹模更新）
```
缺少任何一个，MC本构不完整。

---

### P6. SForceBoundary集合必须为低维度单元

> 已在成功方案S4中详述，此处重申。

- 2D模型：集合只能是 T3D2（线单元），不能是 CPS3/CPS4（面单元）
- 3D模型：集合只能是表面三角形单元，不能是 C3D4 体单元
- 错误集合类型不会静默失败，会报错或默默不起作用

---

### P7. virbox授权问题：必须使用正确版本DLL

> 2026-06-15 反复出现: virbox显示0x0000003连接失败

MultiFracS的 virbox 加密模块与DLL版本绑定：
- 正确版本：`MultiFracS 科研专业版 2026-6-15`
- 错误版本（如2026-05-08）会导致授权失败，程序无法启动
- **解决：** 始终使用 `.exe` 与配套 `.dll` 在同一目录，不混用不同日期的版本

### P8. MultiFracS不能通过PowerShell脚本直接运行（路径空格问题）

条形基础验证中用PowerShell脚本自动化流程，多次因路径含空格（`MultiFracS 科研专业版 2026-6-15`）或中文字符导致脚本报错。

**解决方案：**

### P9. 接触数超限警告（Warning: contact number per element exceeded max）

```
Warning: contact number per element exceeded max (188).
```

此警告不代表结果错误，但说明网格分辨率导致一个单元附近的接触对数超过程序预设上限。
- 影响：可能漏算部分接触对，接触力偏小
- 处理：检查网格是否过密或单元尺寸不均，或用 `*MaxNumPossibleContact` 手动增大上限

---

### P10. 平面应变的正确实现（MultiFracS 3D）

MultiFracS没有原生2D平面应变单元，实现方式：
- 使用C3D4四面体，z方向只保留一层，厚度0.1m
- z方向两侧面（front_z, back_z）施加 `*FixBoundary: vz=0`
- 初始应力需设 `σzz=-30 MPa`（与xy同等），否则平面外应力释放导致不对称
- 监测沿x轴方向布置，提取 `sigma_xx`（≈σr）和 `sigma_yy`（≈σθ）

## 五、Bug记录

### B1. Ce3D本构：应力增量符号约定（2026-03）

**现象：** 早期版本Ce3D出现应力符号翻转问题（压应力显示为正值）

**原因：** 应变增量 `de11` 等由速度梯度计算时，符号约定与主程序不一致

**修复：** 在 `Tet_constitutive_equation_increment` 中验证：拉为正，压为负；标准拉梅方程：
```c
ds11 = 2G*de11 + λ*(de11+de22+de33);
```

---

### B2. 应力量级偏差1000倍问题（Test S1, 2026-03）

**现象：** 施加pn=200000 Pa，MonitorElem读出σxx≈208 Pa（差1000倍）

**当时分析：** 疑似单位系统混用（mm vs m），或面积计算错误

**排查结论：** 50mm×50mm×50mm 盒子面积为 0.0025 m²，pn=200kPa 时面力 = 500 N，体积内应力 = 500/0.05 = 200 kPa。若结果是 200 Pa，说明模型尺寸单位可能是 m 输入了 mm 的数值，即长度被夸大1000倍。

**教训：** inp文件中长度单位必须统一（建议全程用SI：m, Pa, kg）。

---

### B3. 外边界应力维持时SForceBoundary法向方向错误（2026-06）

**现象：** 施加外圆弧面力后，应力反向（变为拉伸）

**原因：** Gmsh导出的面元法向方向可能向内，与预期方向相反

**修复：**
- 在Gmsh中检查Physical Surface法向
- 或在inp中调整pn符号（压为正对应法向向内时应取正值）
- 与官方算例Cylindricalhole3D的 `*SForceBoundary=wai 4,20e6,...` 对齐（itype=4表示法向压力）

---

### B4. MultiFracS每步只输出一次（OutResultFileFrequency设置错误）

**现象：** 计算结束后只有最终帧VTK，无法制作动画，也无法判断中间状态

**原因：** `*OutResultFileFrequency 500000` 与总步数相同

**修复：** 改为
```
*OutResultFileFrequency
5000
```
或根据计算时间调整合适的输出频率。
